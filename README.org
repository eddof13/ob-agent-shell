#+TITLE: ob-agent-shell
#+AUTHOR: Eddie Jesinsky

Org-babel backend for [[https://github.com/xenodium/agent-shell][agent-shell]]. Execute ~#+begin_src agent-shell~ blocks
directly against your running agent-shell buffer — no separate API key
configuration, no second client.

* Why

If you already use agent-shell as your AI client in Emacs, there's no reason
to configure a separate connection just for org-babel blocks. This package
routes babel blocks through your existing agent-shell session.

* Requirements

- Emacs 29.1+
- [[https://github.com/xenodium/agent-shell][agent-shell]] 20260424 or later (requires ~agent-shell-goto-last-interaction~ and ~agent-shell-interaction-at-point~)
- org-mode 9.6+

* Installation

** Manual

Clone this repo and add to your load path:

#+begin_src emacs-lisp
(add-to-list 'load-path "/path/to/ob-agent-shell")
(require 'ob-agent-shell)
(add-to-list 'org-babel-load-languages '(agent-shell . t))
#+end_src

** use-package + straight

#+begin_src emacs-lisp
(use-package ob-agent-shell
  :straight (:host github :repo "eddof13/ob-agent-shell")
  :after (agent-shell org)
  :config
  (add-to-list 'org-babel-load-languages '(agent-shell . t)))
#+end_src

* Usage

Start an agent-shell session first (~M-x agent-shell~), then execute blocks
with ~C-c C-c~:

#+begin_src org
,#+begin_src agent-shell
What is the capital of France?
,#+end_src

,#+RESULTS:
:RESULTS:
Paris.
:END:
#+end_src

Multiline queries work as expected:

#+begin_src org
,#+begin_src agent-shell
Summarize the following in one sentence:

Org-mode is a major mode for Emacs for keeping notes, maintaining
to-do lists, planning projects, and authoring documents.
,#+end_src
#+end_src

* Header arguments

| Argument         | Description                                              |
|------------------+----------------------------------------------------------|
| ~:session NAME~  | Route all blocks sharing NAME to the same agent-shell    |
|                  | buffer. On first use, NAME is bound to the currently     |
|                  | active buffer; subsequent blocks reuse it as long as it  |
|                  | remains live. Set file-wide via ~#+PROPERTY~.            |
| ~:buffer NAME~   | Use a specific agent-shell buffer by name. Takes         |
|                  | priority over ~:session~.                                |
| ~:timeout N~     | Override ~ob-agent-shell-timeout~ for this block only.   |
|                  | N is seconds. Useful for long-running prompts without    |
|                  | raising the global default.                              |
| ~:results drawer~ | Wrap result in ~:RESULTS:...:END:~ tokens so org-babel   |
|                  | can reliably find and replace it on re-run. Default.     |
| ~:results raw~   | Insert result as raw org without a drawer wrapper.       |
|                  | Results may append instead of replace on re-run.         |
| ~:exports both~  | Include both the source block and result on export.      |
|                  | This is the default.                                     |

** Persistent sessions per file

Use ~:session~ to give all blocks in a file their own shared shell — analogous
to a Jupyter kernel per notebook. Set it once via ~#+PROPERTY~ and every block
in the file inherits it:

#+begin_src org
,#+PROPERTY: header-args:agent-shell :session my-session :results drawer
#+end_src

On the first block execution, the session name is bound to the currently active
agent-shell buffer. Every subsequent block with the same name routes to that
same buffer, preserving conversation context across all blocks.

A natural pattern for interactive book or document notes: use a short,
descriptive session name so each file gets its own isolated shell:

#+begin_src org
,#+PROPERTY: header-args:agent-shell :session my-book :results drawer

,#+name: load-book
,#+begin_src agent-shell :timeout 120
The book for this session is at: ~/Downloads/my-book.pdf
Read the PDF in full before responding to any subsequent prompts.
,#+end_src

,#+begin_src agent-shell
Summarize the author's central argument in two paragraphs.
,#+end_src
#+end_src

If the session buffer dies (e.g. after restarting Emacs), the stale entry is
evicted automatically and the next block re-binds the name to whatever buffer
is currently active.

Individual blocks can still override session routing with ~:buffer~ — ~:buffer~
always takes priority over ~:session~.

** Per-block timeout

Use ~:timeout N~ to raise the timeout for a single block without changing the
global default.  Useful for prompts that require significant processing time,
like reading a full PDF before answering questions:

#+begin_src org
,#+name: load-book
,#+begin_src agent-shell :timeout 120
Read the PDF at ~/Downloads/book.pdf in full before responding.
,#+end_src
#+end_src

The global ~ob-agent-shell-timeout~ (default 30s) still applies to all other
blocks.

** Result format

Results are wrapped in ~:RESULTS:...:END:~ by default (~:results output drawer~),
which lets org-babel reliably find and replace them on re-run:

#+begin_src org
,#+begin_src agent-shell
List three fruits, one per line.
,#+end_src

,#+RESULTS:
:RESULTS:
Apple
Banana
Cherry
:END:
#+end_src

Use ~:results raw~ to insert the result as plain org text without a drawer.
This is useful when you want org headings in the results to appear in the
outline tree, but note that re-running the block may append instead of replace:

#+begin_src org
,#+begin_src agent-shell :results raw
Summarize in one paragraph with a * Heading.
,#+end_src
#+end_src

** Targeting a specific buffer

Use ~:buffer~ to route a block to a named agent-shell session:

#+begin_src org
,#+begin_src agent-shell :buffer *claude-shell*
Explain this error...
,#+end_src
#+end_src

To find buffer names: ~M-x list-buffers~ or ~(mapcar #'buffer-name (agent-shell-buffers))~.

*Tip:* keep a dedicated idle buffer for org-babel (e.g. always pass ~:buffer ob-babel~).
~agent-shell-goto-last-interaction~ always jumps to the most recent interaction, so
concurrent turns in the same buffer can cause the wrong response to be captured.


* Customization

#+begin_src emacs-lisp
;; Seconds to wait for a response before timing out (default: 30)
(setq ob-agent-shell-timeout 30)

;; Convert markdown responses to org-mode format via pandoc (default: nil)
;; Requires pandoc: brew install pandoc
(setq ob-agent-shell-convert-markdown t)
#+end_src

** Markdown to org conversion

Agent responses are markdown by default.  With ~ob-agent-shell-convert-markdown~
enabled, pandoc converts responses to org-mode syntax — headers become ~*~
outline levels, code blocks become ~#+begin_src~, links become ~[[url][text]]~,
and so on.

#+begin_src org
,#+begin_src agent-shell
What is the syntax for a for loop in Python?
,#+end_src

,#+RESULTS:
,* Python for loop

,#+begin_src python
for i in range(10):
    print(i)
,#+end_src
#+end_src

* How it works

When a block is executed:

1. Subscribes to ~turn-complete~, ~error~, and ~permission-request~ events
2. Sends the block body via ~agent-shell-insert~
3. Waits via ~sit-for~ (non-freezing — display and timers remain live)
4. On ~turn-complete~, walks the response region in the buffer collecting only
   plain text spans, skipping tool-call UI fragments (~agent-shell-ui-state~)
5. Cleans up all subscriptions and the timeout timer via ~unwind-protect~

If a tool call triggers a permission prompt, a message appears in the echo
area pointing you to the agent-shell buffer to approve it.

* Relationship to agent-shell upstream

This package depends on three public APIs added to agent-shell in version
20260424.1630:

- ~agent-shell-shell-buffer~ — public buffer resolver
- ~agent-shell-goto-last-interaction~ — navigate to the last interaction
- ~agent-shell-interaction-at-point~ — extract the interaction at point

* Known issues

** Org fontification error at source block

Org mode tries to apply syntax highlighting to ~agent-shell~ blocks using a
corresponding major mode, which doesn't exist. This produces a ~Org mode
fontification error~ message. Fix it by mapping the language to ~text-mode~:

#+begin_src emacs-lisp
(add-to-list 'org-src-lang-modes '("agent-shell" . text))
#+end_src

If you use the ~use-package~ installation above, add this line inside the
~:config~ block.
